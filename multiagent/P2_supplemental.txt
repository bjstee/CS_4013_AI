Q1.2:
    My value function is as follows:
    score = CurrentScore + \
            (1 / closestFoodDist) + \
            sumGhosts(
                -> 10 / (ghostDist + 1) if ghost is scared
                -> -100 if ghostDist < 3
                -> -2 / (ghostDist + 1) otherwise
            ) + \
            -numFoodRemaining * 4 + \
            -5 if action == STOP

    We add 1/closestFoodDist to incentivise pacman to prefer a closer food pellet, subtract the numFoodRemaining
    to incentivise choosing a state that results in fewer food (by eating), and we want to flee ghosts if they are
    too close (I chose 3 as the flee distance as this felt right to me). If ghosts are scared, we want to chase
    them, so incentivise as such, and otherwise, we generally want to avoid ghosts.

    Didn't incentivise capsules because it didn't end up being necessary to pass the autograder, but it can be improved
    to do this later.

Q2.1:
    In my algorithm, I define a minimax function to be recursively called. It takes in the current state,
    depth (iterating backwards), and the agent index as input.

    In minimax, we explore all the way down each branch to either a terminal state or the max depth (as is
    often the situation here) and get the utility. This is done by a recursive function call:
    for each node in state tree:
    - If the state is a terminal state, get the utility
    - If the next agent is a max agent, find the max value from it's children
        - minimax(successor, depth - 1, nextAgent)
    - If the next agent is a min agent, find the min value from it's children
        - minimax(successor, depth - 1, nextAgent)

    Becuase we always start with a maximizing agent (pacman is agent 0), we start the call to the minimax function
    by calling a for loop which gets each of the successors of the root state, and does a maximizing calculation on them.
